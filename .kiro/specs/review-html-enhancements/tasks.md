# Implementation Plan

This plan extends the `anime-song-learning-app` review page (parent R8) with four usability features and one packaging fix, sitting on top of a foundational change that pulls every HTML/CSS/JS byte out of Python and into a new static template file. The work lands in strict dependency order: the new `scripts/review_template.html` has to exist before the rewritten `scripts/review.py` can run (review.py reads it on every call), so the template is Task 2 and the Python rewrite is Task 3. Tests update next — the existing `tests/integration/test_review.py` keeps most of its assertions but swaps the raw-HTML greps for JSON-data-block reads and a Python-side DOM simulator. The simulator itself (Task 5) lands before the new property tests (Task 7) that depend on it. Packaging (Task 8) is an independent edit to `tools/package.py`. The final task is `make clean && make test` at ≥ 90% line coverage on `scripts/`.

Every sub-task maps to one or more specific R-RH-N clauses from `requirements.md` and to a named section of `design.md`. Sub-tasks marked with `*` are optional: they improve developer quality of life or ship small UX sugar but are not required to hit the 90% coverage target or satisfy any acceptance criterion. The runtime still imports only the Python standard library; the inline JavaScript shipped in the template runs in the user's browser, not on the deploy target. No new docstring, lint, or mypy constraints are introduced — the existing `pyproject.toml` rules apply.
